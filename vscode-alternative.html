Why switch from VS Code to DL Code?

If you're happy with VS Code, you might wonder why you'd switch. The short answer: AI is built in, not bolted on. VS Code requires extensions for AI features, and the best ones (GitHub Copilot, Cursor) cost money. DL Code ships with a full AI assistant that works with free local models via Ollama — no subscription, no API key, no usage limits.

What's the same as VS Code

  • Same editor engine — DL Code uses Monaco, the same editor that powers VS Code. Your muscle memory works.
  • Same keyboard shortcuts — Ctrl+Shift+P for command palette, Ctrl+P for quick open, Ctrl+B to toggle sidebar, etc.
  • Same file concepts — workspace folders, multi-cursor, find/replace, code folding, minimap.
  • Similar extension model — DL Code has a VS Code-compatible extension API.
  • Same language support — 60+ languages with syntax highlighting.

What's different (and better)

  • AI is built in — no need to install Copilot or Cursor. Claude AI chat is one click away.
  • Agentic app builder — describe an app in plain English, DL Code scaffolds the entire project.
  • Free unlimited AI — via Ollama. Run models locally, no API key, no rate limits.
  • JetBrains-inspired UI — darker, denser, more focused. Less chrome, more code.
  • No telemetry by default — your code stays on your machine.
  • Real terminal — PTY-backed, not a fake console.

Migrating from VS Code to DL Code

Switching is easy. Your files and projects work exactly the same — just open your workspace folder in DL Code and start editing. Most VS Code keyboard shortcuts work identically. The main differences are UI density (DL Code is denser, like JetBrains) and the AI panel (which is built in, not an extension).
